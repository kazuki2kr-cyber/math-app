## ダッシュボード設計原則

### 1. 階層構造（ドリルダウン設計）
- **概要レベル**: 全体のKPIサマリー（DAU、総演習回数、全体正答率）
- **詳細レベル**: 単元別、ユーザー別の深堀り分析
- **個別レベル**: 特定の問題や特定のユーザーの詳細データ

### 2. 教育アプリ向けKPI
#### North Star Metrics
- **アクティブ学習率**: 過去7日間に演習を行ったユーザー割合
- **学習効果指標**: ユーザー全体の正答率改善度

#### Supporting Metrics
- **DAU/WAU/MAU**: 日次/週次/月次アクティブユーザー
- **単元完了率**: 各単元の演習完了ユーザー数
- **リテンション率**: 1日, 7日, 30日リテンション
- **難易度分布**: 正答率による問題の難易度バランス

### 3. 可視化パターン
- **サマリーカード**: 主要KPIをカード形式で一覧表示（前期比較付き）
- **トレンドチャート**: 時系列の変化をエリア/線グラフで
- **ヒートマップ**: 相関行列、活動パターン（曜日×時間帯）
- **バーチャート**: カテゴリ比較（単元別正答率等）
- **テーブル**: 詳細データの一覧にはソート・フィルタ付きテーブル

### 4. グラフライブラリ選定
Next.js/React環境では以下を推奨:
- **Recharts**: React向け最も統合が容易。SSR対応。軽量。
- **Chart.js + react-chartjs-2**: 汎用的で設定自由度が高い
- 注: D3.jsは高機能だがReact統合が複雑。避ける。

### 5. パフォーマンス最適化
- データの初回取得後はクライアントサイドでの計算を優先
- 重い計算はWeb Workerへオフロードを検討
- 仮想化テーブル（大量行表示時）
