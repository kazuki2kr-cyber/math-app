# Kanji CSV Creation Skill

PDFや画像データとして提供される漢字のテストや問題集から、本システム（math.app 漢字ドリルモジュール）にインポート可能な形式のCSVデータを抽出・作成するための標準ワークフローとルールです。

## 1. 目的（Goal）
画像やPDFから文字を正確に読み取り（OCR）、指定されたCSVスキーマに準拠したデータを生成する。

## 2. 必須データ形式（CSV Schema）
出力するCSVは以下のヘッダーを**必ず**含める必要があります。

```csv
unit_id,title,question_text,answer,explanation
```

### 2.1 カラムの詳細
*   **`unit_id`** (String): 単元の識別子。（例: `"4級第1回"`, `"4級第2回"`）
    *   ※ 特別な指示がない限り、同一リクエスト内の問題はすべて同じ `unit_id` に設定してください。
*   **`title`** (String): ユーザー向け単元表示名。（例: `"4級第1回"`）
    *   ※ 基本的に `unit_id` と同じ値を設定して構いません。
*   **`question_text`** (String): 問題文。
    *   **重要**: 解答となる漢字部分は**カタカナ**にし、前後にHTMLの `<u>` タグをつけて下線を引く表現にしてください。
    *   （例: `"彼の<u>ケンジツ</u>な仕事ぶりには定評がある。"`）
*   **`answer`** (String): 正解となる**漢字**文字列。（例: `"堅実"`）
*   **`explanation`** (String): その漢字（熟語）の端的な意味や解説。（例: `"堅実（けんじつ）は着実でしっかりしていることを意味します。"`）

## 3. CSV生成のルール
1.  **カンマおよび改行の処理**: `question_text` や `explanation` にカンマ（`,`）や改行が含まれる場合は、必ずフィールド全体をダブルクォーテーション（`"`）で囲んでください。安全のため、全テキストフィールドを `"` で囲むスタイルを推奨します。
2.  **正確性の担保**: 長文の画像からテキストを起こす場合、文脈を補完して正確な日本語になるように微調整してください。
3.  **読み仮名への変換**: 問題文の中の（　）や空白で虫食いになっている箇所は、問題の意図を把握した上で、その箇所の解答の**カタカナ読み**をあてはめてください。

## 4. エージェント側での実行手順 (Workflow for AI)
AIエージェントがこのスキルを実行する際は、以下のステップを踏んでください。

1.  **入力の確認**: ユーザーから画像/PDFと、対象となる `unit_id` (および `title`) を受け取ります。
2.  **文字認識と構造化**: 
    *   画像から「問題文」と「本来当てはまるべき漢字」を抽出します。
    *   本来の漢字の意味を簡潔に調べ（またはAI自身の知識から）、`explanation` 用の解説文を生成します。（例: 「〇〇（よみ）は□□という意味です」）
3.  **CSV構築**: 上記 2. で定義したスキーマに従ってCSVデータを作成します。
4.  **出力・ファイル生成**: 
    *   作成したCSVをフロントエンドのUIでコードブロックとして表示するか、可能であれば指定されたファイル（例: `scripts/〇〇.csv`）に直接保存してユーザーに完了を報告します。

## 5. 出力例（Example）

```csv
unit_id,title,question_text,answer,explanation
"4級第2回","4級第2回","彼の<u>ケンジツ</u>な仕事ぶりには定評がある。","堅実","堅実（けんじつ）は着実でしっかりしていることを意味します。"
"4級第2回","4級第2回","言葉の問題が<u>ショウヘキ</u>となる海外生活。","障壁","障壁（しょうへき）はさまたげとなる壁・障害のことです。"
"4級第2回","4級第2回","先生は彼に<u>ゼンプク</u>の信頼をよせている。","全幅","全幅（ぜんぷく）は全く・すべてという意味で使います。"
```
