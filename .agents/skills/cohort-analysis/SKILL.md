## コホート分析の概要
コホート分析は、ユーザーを特定の共通属性（主に登録時期）でグループ化し、時間経過に伴う行動変化を追跡する分析手法です。

## Formixでの適用方法

### コホートの定義
- **時間ベース**: ユーザーの最初のログイン日・週・月でグルーピング
- **行動ベース**: 最初に取り組んだ単元、最初の正答率レベルなどでグルーピング

### 追跡メトリクス
1. **リテンション率**: 登録後N日/週/月後にまだ演習を行っているユーザーの割合
2. **エンゲージメント深度**: コホートごとの平均演習回数、到達単元数
3. **学習成果**: コホートごとの正答率の改善度合い

### 分析用データ構造
```typescript
interface CohortData {
  cohortKey: string;       // "2024-W12" or "2024-03"
  userCount: number;       // コホート内のユーザー数
  retention: number[];     // 各期間のリテンション率 [100, 85, 72, ...]
  avgAccuracy: number[];   // 各期間の平均正答率
  avgDrillCount: number[]; // 各期間の平均演習回数
}
```

### Firebase最適化
- コホートデータは **クライアントサイドで計算** し、Firestoreから取得するのは `users` と `attempts` のみ
- 計算結果は `analytics_cache` コレクションに保存し、再利用可能に
- TTL（有効期限）を設定し、古いキャッシュは自動的に再計算

### 可視化
- リテンション行列（ヒートマップ形式）
- コホート別の学習進捗推移グラフ
