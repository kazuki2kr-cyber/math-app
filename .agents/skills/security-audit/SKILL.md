# Security Audit Skill for Formix

このスキルは、アプリケーションのソースコードや設定に変更を加える際に、エージェント自身が「セキュリティ観点でのセルフレビュー」を実施するためのガイドラインです。

## 1. 差分レビュー (Differential Review)
既存コードから新しいコードへ変更を加える際、以下の観点で差分をレビューしてください。
- **権限バイパスの有無**: 管理者権限（Admin）チェックのロジックがバイパスされていないか？（例: `isAdmin` による条件分岐が意図せず消去されていないか）
- **入力値検証 (Validation)**: クライアントからの入力値（ドリル結果、スコアデータなど）をそのまま信用して処理していないか？ XSS や Injection リスクがないか確認すること。
- **データ露出**: Firestore から取得したデータのうち、本来クライアントに渡すべきでない機密情報（他のユーザーのメールアドレスや詳細な進行データ）をフロントエンドに意図せず露出させていないか？

## 2. 脆弱なデフォルト設定の検知 (Insecure Defaults)
環境構築やパッケージ追加時に、セキュリティリスクのある設定が行われていないか監査します。
- 環境変数のハードコード: 認証情報、APIキー、シークレットトークンなどがコードに直接記述されていないか？（例外: Playwright用のダミーパスワードや公開前提の `NEXT_PUBLIC_` プレフィックスを除く）
- CORS / セキュリティヘッダー設定不足: Next.js の `next.config.ts` で適切なセキュリティヘッダー（`X-Frame-Options`, `Content-Security-Policy` 等）が設定されているか定期的に確認すること。
- 依存関係のバージョン: パッケージ追加時に、既知の脆弱性がある過去バージョン（CVE報告済みなど）を無自覚に追加していないか確認すること。

## 3. デプロイ前の自動チェック要件
デプロイを実行する直前や、プルリクエスト等の区切りの際には、エージェント自身がこのスキルをロードし、変更ファイル全体を対象に「セキュリティレビューを実施した旨」を宣言してから作業を進捗させてください。

## トリガー条件
- 「デプロイの準備をして」「デプロイしたい」
- 「権限設定を変更して」「認証フローを追加して」
- 「新しいAPIルート（Cloud Functions）を作成して」
- 定期的な「コード監査をお願いします」などの発言時。
