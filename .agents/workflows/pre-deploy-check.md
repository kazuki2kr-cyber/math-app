# デプロイ前チェックフロー

デグレを防止し、アプリケーションの品質を維持するために、以下の手順を**必ず**実行してください。

## 1. セキュリティルールの自動テスト
セキュリティルール (`firestore.rules`) を変更した際や、定期的なデプロイ前には必ず以下のコマンドを実行し、権限設定に不備がないか確認してください。

// turbo
```bash
npm test tests/firestore.rules.spec.ts
```

- すべてのテストが「Passed」になることを確認してください。
- ルールを変更した場合は、必要に応じて `tests/firestore.rules.spec.ts` に新しいテストケースを追加してください。

## 2. 自動E2Eテスト (Playwright)の実行
手動での確認の代わりに、クリティカルパス（ログイン 〜 ドリル実行 〜 XP付与）の E2E テストを実行します。

// turbo
```bash
npm run test:e2e
```

- テストがすべて「Passed」になることを確認してください。
- 失敗した場合は、Playwright のレポートを確認し、ダッシュボードや関数のエラー原因を追及してください。

## 3. エージェントによるセキュリティ監査（セルフレビュー）
変更箇所に対して、`security-audit` スキルを活用して以下の宣言・チェックを実施してください。

- **指示例**: 「デプロイ前に security-audit スキルを使って、今回の差分に脆弱性や権限設定漏れがないかチェックして」
- **確認事項**:
  - `admin` 権限などのバイパスがないか。
  - 生徒からの意図しないデータ更新リスクがないか。

## 3. Cloud Functions のログ確認
ドリル完了後に `functions status` や Firebase Console のログを確認し、`processDrillResult` でエラーが発生していないかチェックしてください。
