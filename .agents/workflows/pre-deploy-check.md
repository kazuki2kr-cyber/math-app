# デプロイ前チェックフロー

デグレを防止し、アプリケーションの品質を維持するために、以下の手順を**必ず**実行してください。

## 1. セキュリティルールの自動テスト
セキュリティルール (`firestore.rules`) を変更した際や、定期的なデプロイ前には必ず以下のコマンドを実行し、権限設定に不備がないか確認してください。

// turbo
```bash
npm test tests/firestore.rules.spec.ts
```

- すべてのテストが「Passed」になることを確認してください。
- ルールを変更した場合は、必要に応じて `tests/firestore.rules.spec.ts` に新しいテストケースを追加してください。

## 2. 実機環境でのスモークテスト
Firebase エミュレータまたはステージング環境で、テスト用生徒アカウントを使用して以下の項目を確認してください。

1. **ログイン**: 正常にログインでき、ダッシュボードが表示されること。
2. **ランキング**: ランキングが正常に表示され、他人の情報取得でエラー（Permission Denied）が出ていないこと。
3. **ドリルの実行**: 1つ以上のユニットを完了させ、リザルト画面が表示されること。
4. **データの反映**: 完了後、XPやスコアがダッシュボードに正しく反映されていること。

## 3. Cloud Functions のログ確認
ドリル完了後に `functions status` や Firebase Console のログを確認し、`processDrillResult` でエラーが発生していないかチェックしてください。
