# 開発ガイドライン & 運用ルール

このドキュメントでは、math-app の品質とセキュリティを維持するための必須ルールを定義します。

## 1. セキュリティの維持 (Regression Prevention)

セキュリティルール (`firestore.rules`) は、学生によるデータの直接書き換えを防ぐための生命線です。

- **ルールの変更**: `firestore.rules` を変更した場合は、必ず対応するテストを `tests/firestore.rules.spec.ts` に追加してください。
- **自動テストの実行**: デプロイ前には必ず以下のコマンドを実行してください。
  ```bash
  npm run test:security
  ```
- **原則**: 学生による `scores`、`users` (XP等)、`stats` への直接書き込みは禁止です。データ更新は必ず Cloud Functions (`processDrillResult`) を経由させてください。

## 2. デプロイ前チェックリスト

デプロイ前には、テスト用生徒アカウントを使用して、以下の機能を**エミュレータまたはステージング環境**で手動または自動で確認してください。

1. **ログインとダッシュボード**:
   - 生徒アカウントでログインし、自分のXP、レベル、ランキングが正常に表示されること。
   - 他人の詳細プロフィール読み取りエラー（コード上のPermission Denied）が発生していないこと。
2. **ドリルの実行と結果保存**:
   - ドリルを完了し、リザルト画面でスコアと獲得XPが表示されること。
   - ダッシュボードに戻った際、XPが増加し、ランキングが更新されていること。

## 3. 自動E2Eテスト環境 (Playwright)

クリティカルパス（ログイン → ドリル実行 → XP確認）を守るため、**Playwright** を用いたE2Eテストが導入されています。

- **実行コマンド**:
  ```bash
  npm run test:e2e
  ```
- **環境設定**:
  `.env.local` に `TEST_USER_EMAIL` と `TEST_USER_PASSWORD` が設定されていることを確認してください（パスワードはテスト専用のものを指定し、Gitにはコミットしないでください）。
- **CI / CD 等の自動化**:
  デプロイ前には手動あるいはワークフロー経由で必ず実行し、デグレが発生していないことを保証してください。

## 4. エージェントの自律化スキル・ルール

今後の開発・デバッグ・デプロイにおいて、エージェント（Antigravity等）自身が以下のルールを自律的に遵守します。

- **Firebase / Firestore**:
  `.agents/skills/firebase-best-practices` に定義されたルールに従い、ルールの厳格化や Cloud Functions でのトランザクション処理を自動でレビュー・実装します。
- **セキュリティ・監査**:
  `.agents/skills/security-audit` に定義されたルール（Trail of Bits相当）に従い、デプロイや権限変更の前に、脆弱な設定がないか、入力値検証が抜けていないかの差分レビューを自立的に実施します。
- **React / Next.js UI設計**:
  UIの改修時は、標準搭載の `vercel-react-best-practices` および `vercel-composition-patterns` スキルを適用し、パフォーマンスと保守性の高いコンポーネント設計を行います。
