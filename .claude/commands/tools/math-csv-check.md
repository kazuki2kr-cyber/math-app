# /tools:math-csv-check — 数学問題CSVの検証・修正

数学問題CSVファイルの内容を検証し、数学的・形式的な誤りを修正するスキルです。

---

## 実行手順

### Step 1: 対象CSVの特定

ユーザーが指定したCSVファイル（または `data/csv/` 以下のCSVファイル群）を対象とする。

### Step 2: 自動検証スクリプトの実行

`validate_csv.py` を使って基本的な形式チェック・自動修正を行う。

```bash
python .agents/skills/pdf-to-math-csv/scripts/validate_csv.py <対象CSVパス>
```

複数ファイルを一括チェックする場合:

```bash
for f in data/csv/**/*.csv; do
  python .agents/skills/pdf-to-math-csv/scripts/validate_csv.py "$f"
done
```

スクリプトが自動修正する内容:
- 先頭ゼロの除去（例: `07` → `7`）
- `options` が正しいJSON配列（4要素）かチェック・修正
- `answer_index` が 1〜4 の整数かチェック
- LaTeXデリミタ `\( ... \)` の開閉不一致を警告

### Step 3: 詳細チェック（Claude が全行を精読して実施）

スクリプトで検出できない問題を、以下のカテゴリに従って1行ずつ検証する。

---

#### A. 数値・整数表現

| チェック | 例（誤） | 例（正） |
|---------|---------|---------|
| A1. 先頭ゼロ（整数） | `01`, `02`, `03` | `1`, `2`, `3` |
| A2. 不要な末尾ゼロ（結果が整数の小数） | `8.0`, `-15.0`, `-7.0` | `8`, `-15`, `-7` |
| A3. `\frac{0}{n}` 表記 | `\frac{0}{3}` | `0` |
| A4. `\frac{n}{1}` 表記 | `\frac{5}{1}` | `5` |

---

#### B. 分数の既約・表記

| チェック | 例（誤） | 例（正） |
|---------|---------|---------|
| B1. 既約でない分数（gcd > 1） | `\frac{6}{9}`, `\frac{4}{2}` | `\frac{2}{3}`, `2` |
| B2. 分子にマイナス符号 | `\frac{-5}{8}` | `-\frac{5}{8}` |
| B3. 分母にマイナス符号 | `\frac{5}{-8}` | `-\frac{5}{8}` |
| B4. 分子・分母の両方にマイナス符号 | `\frac{-5}{-8}` | `\frac{5}{8}` |
| B5. 分子＝分母（= 1）の分数 | `\frac{7}{7}`, `\frac{9}{9}` | `1` |

---

#### C. 問題文中の不適切な分数

問題文（`question_text`）に登場する分数も確認する。ただし**問題の計算内容を変える修正はしない**。

- 修正してよいケース：値が 1 になる分数（`\frac{n}{n}` → `1`）
- 修正しないケース：同分母の加減算問題で分母を揃えて提示している分数（例：`\frac{6}{2} - \frac{1}{2}` の `\frac{6}{2}`）

---

#### D. LaTeX・記号表記

| チェック | 例（誤） | 例（正） |
|---------|---------|---------|
| D1. アポストロフィが数式に混入 | `3'`, `2^4 \times 3'` | `3`, `2^4 \times 3` |
| D2. 等式・不等式に全角文字 | `＝`, `＞`, `＜` | `=`, `>`, `<` |
| D3. 乗算記号の混在（LaTeX内） | `×` と `\times` が混在 | `\times` に統一 |

---

#### E. 選択肢の構造的問題

| チェック | 内容 |
|---------|------|
| E1. 重複選択肢（同じ値の別表記） | `\frac{-3}{2}` と `\frac{3}{-2}` は同値。正解でない側を別の誤答に差し替える |
| E2. 選択肢が正解と同値（別表記） | 正解 `-\frac{3}{4}` なのに `\frac{-3}{4}` も選択肢にある場合 |
| E3. 選択肢の数が4つでない | 通常は4択。不足・過剰があれば指摘 |
| E4. 代数的同値（見た目が違っても数学的に等しい式） | 正解 `a^2b^2` なのに `(ab)^2` も選択肢にある場合。下記の変換パターンを確認する |

**E4 で確認すべき代数的同値パターン（中学数学の範囲）**

| パターン | 同値の例 |
|---------|---------|
| 累乗の展開 | `(ab)^2` = `a^2b^2`、`(2x)^3` = `8x^3` |
| 指数法則 | `a^2 \times a^3` = `a^5`、`(x^2)^3` = `x^6` |
| 分配法則の展開 | `2(x+3)` = `2x+6` |
| 因数分解形 | `x^2-1` = `(x+1)(x-1)`（中2以降） |
| 符号の移動 | `-\frac{x}{7}` = `\frac{-x}{7}` = `\frac{x}{-7}`（B2〜B4 と重複） |
| 係数1の省略 | `1 \times x^2` = `x^2`、`-1 \times ab` = `-ab` |
| 分数の別表記 | `\frac{3a}{4}` と `0.75a`（有理数係数の場合） |
| 順序違いの単項式 | `5b^2a` = `5ab^2`（アルファベット順でないだけで同じ式） |

正解と同値の選択肢が見つかった場合は差し替える。差し替え候補は「正解とは異なる」「他の選択肢とも異なる」「生徒がよく犯す計算ミス」を基準に選ぶ。

重複・同値の差し替えは「正解と値が異なる」「他の選択肢と値が異なる」「生徒がよく犯す計算ミス（符号ミス・約分ミス・逆数間違い等）」を基準に選ぶ。

---

#### F. answer_index の整合性

- `answer_index` は **1始まり**（1〜4）であることを確認
- `answer_index` が指す選択肢テキストと `explanation` 内の「正解は〜」が一致しているか確認
- 簡単な計算（加減乗除・約分・素因数分解）は実際に計算して answer_index の正誤を検証する

---

#### G. explanation との整合性

- `explanation` 内の正解表記が B2〜B5 のルールに違反していないか確認して修正する
- 例：`正解は \( \frac{-1}{7} \) です` → `正解は \( -\frac{1}{7} \) です`

---

#### H. 文章・論理の整合性

以下の観点でテキストを精読し、問題として成立しているかを確認する。修正できるものは修正し、判断が難しいものは指摘のみとする。

| チェック | 内容 |
|---------|------|
| H1. 問題文と選択肢の型が一致 | 「次の数の絶対値を答えなさい」→ 選択肢が数値でない場合は指摘 |
| H2. 問題文に計算式・条件が存在する | 「次の計算をしなさい」なのに式が抜けている場合は指摘 |
| H3. 選択肢が問いに対して意味をなす | 単位・形式が問題と一致しているか（例：問いが「℃」なのに選択肢が「m」） |
| H4. 解説が正解の導き方を正しく説明している | 解説の計算過程が正解に一致しているか |
| H5. 解説内の計算・数値に誤りがない | 解説中の中間計算が正しいか（例：`\( -14 + (-3) = -18 \)` → 誤り、正しくは `-17`） |
| H6. 選択肢の日本語表現が自然か | 「0から遠い数」など意味が曖昧・不適切な表現がないか |

---

#### I. 数学的正誤検証

`answer_index` が指す選択肢が数学的に正しい答えかを検証する。

**計算問題（四則・分数・累乗・素因数分解）**：実際に計算して一致を確認する。不一致の場合は正しい `answer_index` を特定し、問題として報告する（**自動修正はせず、必ず人間に確認を求める**）。

**文章題・概念問題**：設定されている数量関係や定義と、正解とされる選択肢が矛盾しないかを推論で確認する。

**検証できない場合**：判断不能として理由とともに指摘する。

---

#### J. 画像対応確認

`image_url` が空でない行に対して実施する。空の行はスキップする。

1. WebFetch で `image_url` の画像を取得する
2. 取得できた場合：画像の内容と `question_text` の内容が対応しているかを確認する
   - 問題文が「右の図を見て答えなさい」→ 画像が図として存在するか
   - 画像に描かれた数値・図形・グラフが問題文の設定と一致しているか
3. 取得できなかった場合（認証エラー・404 等）：URLのみ記録し「手動確認が必要」と報告する

---

#### K. LaTeX 記法チェック

このアプリの表示ルールを前提として確認する。

**列ごとのエスケープルール（最重要）**

| 列 | 種別 | バックスラッシュ | デリミタ例 |
|----|-----|--------------|---------|
| `question_text` | プレーンテキスト | 1本 `\` | `\( \frac{3}{2} \)` |
| `options` | JSON配列文字列 | 2本 `\\` | `["\\( \\frac{3}{2} \\)", ...]` |
| `explanation` | プレーンテキスト | 1本 `\` | `正解は \( \frac{3}{2} \) です` |

**アプリが認識する数式デリミタ（MathDisplay コンポーネント）**

- `\( ... \)` → インライン数式
- `\[ ... \]` → ブロック数式
- `$ ... $` → インライン数式
- `$$ ... $$` → ブロック数式
- デリミタなし + バックスラッシュあり + 日本語なし → フォールバックでインライン数式として描画（動作するが非推奨）

| チェック | 内容 |
|---------|------|
| K1. JSON エスケープ（options列） | `options` 列のJSON文字列内でバックスラッシュが1本（`\frac`）になっていないか。正しくは `\\frac`。JSON.parse が失敗するか、描画が壊れる |
| K2. デリミタの対応 | `\(` に対して `\)` が存在するか。`\[` に対して `\]` が存在するか。開き・閉じの不一致を検出する |
| K3. デリミタの入れ子 | `\( \( ... \) \)` のような入れ子はエラーになる |
| K4. options内の形式統一 | 同一ファイル内で `\\( \\frac{} \\)` 形式と `\\frac{}` 形式が混在していないか。混在は許容するが報告する |
| K5. よくあるLaTeXコマンドのスペルミス | `\farce`, `\tmes`, `\sqt` 等のタイポ。正しいKaTeXコマンドかを確認する。中学数学で使われる主要コマンド：`\frac`, `\times`, `\div`, `\sqrt`, `\pm`, `\cdot`, `\leq`, `\geq`, `\neq`, `\approx`, `\infty`, `\text{}`, `^`, `_` |
| K6. 空の数式デリミタ | `\( \)` のように中身が空の数式デリミタはエラーになる |
| K7. 中括弧の対応 | `{` と `}` の個数が一致しているか（`\frac{3}{2` のような閉じ忘れ） |
| K8. 全角文字の数式内混入 | `\( ３÷２ \)` のように全角数字・全角演算子が数式デリミタ内に入っていないか |

---

### Step 4: 修正の実施

問題が見つかった場合はファイルを直接編集して修正する。

**自動修正してよい項目**: A, B, C（修正可ケースのみ）, D, E（同値重複の差し替え）, G, H（修正できるもの）, K

**必ず人間に確認を求める項目**: I（数学的正誤が不一致の場合）

修正内容をユーザーに報告する際は以下の形式で:

```
【修正レポート】ファイル名: xxx.csv
---
行X [チェックID]: （何が間違っていたか）
  修正前: ...
  修正後: ...
---
【要確認】（自動修正せず人間に確認が必要なもの）
行X [I]: answer_index が数学的に不一致の疑い
  問題: ...
  現在の answer_index: N（選択肢: ...）
  計算結果: ...（別の選択肢が正解の可能性）
---
合計 N 件を修正、M 件を要確認としてマークしました。
```

---

### Step 5: 最終確認

修正後、以下の3点を再確認する:
1. 問題と正解の整合性（F・I の再検証）
2. 選択肢の重複チェック（E）
3. データの重複行がないか（同じ問題が2行以上ないか）

---

## 注意事項

- `answer_index` は **1始まり**（1〜4）
- `options` のJSON内バックスラッシュ: CSVファイル上は `\\` 1本（例: `"\\frac{1}{2}"`）
- `question_text`/`explanation` 内バックスラッシュ: `\(` と `\)` で囲む（例: `\( \frac{1}{2} \)`）
- スクリプトの自動修正後に必ず Step 3 の人力チェックを行うこと
- 解説文に AI の独白（「あれ？」「すみません」「計算し直します」等）が入っていないか確認する
